## MODIFIED Requirements

### Requirement: 笔记管理
系统应提供一个功能丰富的笔记管理系统，支持 Markdown 编辑、文件组织、标签分类、版本历史和媒体展示。

#### Scenario: 创建新笔记
- **WHEN** 用户点击"新建笔记"或使用 `@note` 命令
- **THEN** 系统创建新笔记
- **AND** 打开 Markdown 编辑器
- **AND** 将笔记保存为 `.md` 文件到配置的存储位置
- **AND** 支持从模板创建笔记

#### Scenario: 编辑笔记
- **WHEN** 用户从文件树中选择笔记
- **THEN** 系统将笔记内容加载到编辑器
- **AND** 用户可以编辑 Markdown 内容
- **AND** 更改自动保存或手动保存
- **AND** 自动保存历史版本

#### Scenario: 删除笔记
- **WHEN** 用户右键点击笔记并选择"删除"
- **THEN** 系统提示确认
- **AND** 确认后删除笔记文件
- **AND** 从文件树中移除

#### Scenario: 创建文件夹
- **WHEN** 用户在文件树中右键点击并选择"新建文件夹"
- **THEN** 系统创建新文件夹
- **AND** 用户可以命名文件夹
- **AND** 文件夹出现在文件树中
- **AND** 支持多级文件夹嵌套

#### Scenario: 在文件夹中组织笔记
- **WHEN** 用户将笔记拖拽到文件夹
- **THEN** 系统将笔记文件移动到文件夹目录
- **AND** 更新文件树结构

#### Scenario: 使用标签管理笔记
- **WHEN** 用户编辑笔记
- **THEN** 系统允许添加多个标签
- **AND** 标签自动建议（基于已有标签）
- **AND** 支持按标签筛选笔记
- **AND** 显示标签云

#### Scenario: 使用分类管理笔记
- **WHEN** 用户编辑笔记
- **THEN** 系统允许选择分类
- **AND** 支持多级分类
- **AND** 分类可设置颜色标记
- **AND** 支持按分类筛选笔记

#### Scenario: 使用笔记模板
- **WHEN** 用户创建新笔记
- **THEN** 系统提供模板选择
- **AND** 包括会议记录、项目计划、读书笔记等模板
- **AND** 支持自定义模板
- **AND** 从模板创建笔记时填充初始内容

#### Scenario: 查看版本历史
- **WHEN** 用户查看笔记
- **THEN** 系统显示版本历史列表
- **AND** 包括保存时间和修改摘要
- **AND** 支持版本对比
- **AND** 支持恢复到历史版本

#### Scenario: 全文搜索笔记
- **WHEN** 用户在搜索框中输入关键词
- **THEN** 系统搜索所有笔记内容
- **AND** 高亮显示搜索结果
- **AND** 支持搜索历史记录
- **AND** 支持高级搜索（标题、内容、标签组合）

#### Scenario: 导出笔记
- **WHEN** 用户选择导出笔记
- **THEN** 系统支持导出为 PDF
- **AND** 支持导出为 HTML
- **AND** 支持批量导出为 ZIP 压缩包
- **AND** 保持格式和样式

## MODIFIED Requirements

### Requirement: Markdown 编辑器
系统应提供一个功能强大且美观的 Markdown 编辑器，支持编辑/预览切换、语法高亮、拖拽粘贴和扩展功能。

#### Scenario: 编辑 Markdown 内容
- **WHEN** 用户在编辑器中输入
- **THEN** 系统支持标准 Markdown 语法
- **AND** 提供语法高亮
- **AND** 使用优化的字体、行高和间距
- **AND** 提供清晰的视觉反馈
- **AND** 支持代码块语法高亮
- **AND** 支持表格编辑器
- **AND** 支持数学公式（LaTeX）
- **AND** 支持思维导图

#### Scenario: 切换编辑和预览模式
- **WHEN** 用户点击编辑/预览切换按钮
- **THEN** 系统在编辑模式和预览模式之间切换
- **AND** 编辑模式显示 Markdown 编辑器
- **AND** 预览模式显示渲染后的 HTML
- **AND** 两种模式不并排显示（单屏切换）
- **AND** 切换流畅无闪烁

#### Scenario: 美化编辑器界面
- **WHEN** 用户在编辑模式下
- **THEN** 编辑器使用优化的字体和行高
- **AND** 提供舒适的间距和边距
- **AND** 支持代码语法高亮
- **AND** 提供清晰的视觉反馈
- **AND** 优化预览样式（更好的排版、代码高亮）

#### Scenario: 使用键盘快捷键
- **WHEN** 用户在编辑模式下
- **THEN** 系统支持 Ctrl+S 保存笔记
- **AND** 支持 Tab 键插入缩进
- **AND** 支持其他常用快捷键

#### Scenario: 插入图片
- **WHEN** 用户通过按钮或菜单插入图片
- **THEN** 系统将图片保存到图片文件夹
- **AND** 使用相对路径更新 Markdown
- **AND** 在预览中显示图片

#### Scenario: 拖拽图片到编辑器
- **WHEN** 用户从文件管理器拖拽图片文件到编辑器
- **THEN** 系统自动检测拖拽操作
- **AND** 自动保存图片到笔记资源目录
- **AND** 自动插入 Markdown 图片语法到光标位置
- **AND** 显示上传进度（如需要）
- **AND** 支持常见图片格式（PNG、JPG、GIF、WebP、SVG 等）

#### Scenario: 拖拽视频到编辑器
- **WHEN** 用户从文件管理器拖拽视频文件到编辑器
- **THEN** 系统自动检测拖拽操作
- **AND** 自动保存视频到笔记资源目录
- **AND** 自动插入 HTML 视频标签到光标位置
- **AND** 显示上传进度（如需要）
- **AND** 支持常见视频格式（MP4、WebM、OGG、MOV、AVI 等）

#### Scenario: 粘贴图片到编辑器
- **WHEN** 用户复制图片后按 Ctrl+V 粘贴到编辑器
- **THEN** 系统检测剪贴板中的图片数据
- **AND** 自动保存图片到笔记资源目录
- **AND** 自动插入 Markdown 图片语法到光标位置
- **AND** 支持从浏览器、截图工具等复制的图片

#### Scenario: 插入视频
- **WHEN** 用户通过按钮或菜单插入视频
- **THEN** 系统将视频保存到视频文件夹
- **AND** 使用相对路径更新 Markdown
- **AND** 在预览中显示视频播放器
