## MODIFIED Requirements

### Requirement: 待办管理
系统应提供功能丰富的待办管理系统，支持多种视图、任务依赖、时间追踪和批量操作。

#### Scenario: 创建待办
- **WHEN** 用户点击"新建待办"或使用 `@todo` 命令
- **THEN** 系统打开表单，包含字段：标题、描述、截止日期、优先级、分类、依赖任务
- **AND** 保存时，将待办存储到 SQLite 数据库
- **AND** 在列表中显示待办
- **AND** 支持从模板创建

#### Scenario: 查看待办列表
- **WHEN** 用户导航到待办
- **THEN** 系统显示所有待办
- **AND** 显示标题、截止日期、优先级和完成状态
- **AND** 支持按状态、日期、优先级过滤
- **AND** 支持多种视图（列表、看板、甘特图）

#### Scenario: 查看看板视图
- **WHEN** 用户切换到看板视图
- **THEN** 系统按状态分组显示（待办/进行中/已完成）
- **AND** 支持拖拽移动任务
- **AND** 支持自定义列

#### Scenario: 查看甘特图视图
- **WHEN** 用户切换到甘特图视图
- **THEN** 系统显示任务时间线
- **AND** 显示任务依赖关系
- **AND** 显示关键路径

#### Scenario: 完成待办
- **WHEN** 用户勾选完成复选框
- **THEN** 系统将待办标记为已完成
- **AND** 更新进度百分比
- **AND** 移动到已完成部分（如果过滤）
- **AND** 自动完成依赖的后续任务（如果配置）

#### Scenario: 编辑待办
- **WHEN** 用户在待办上点击"编辑"
- **THEN** 系统打开包含当前值的编辑表单
- **AND** 用户可以修改任何字段
- **AND** 可以设置任务依赖
- **AND** 更改保存到数据库

#### Scenario: 删除待办
- **WHEN** 用户在待办上点击"删除"
- **THEN** 系统提示确认
- **AND** 确认后从数据库移除待办
- **AND** 支持批量删除

## MODIFIED Requirements

### Requirement: 子任务
系统应支持为待办创建子任务，并自动计算进度。

#### Scenario: 添加子任务
- **WHEN** 用户在待办上点击"添加子任务"
- **THEN** 系统创建新子任务
- **AND** 子任务与父待办关联
- **AND** 在父项下缩进显示
- **AND** 自动更新父任务进度

#### Scenario: 完成子任务
- **WHEN** 用户完成子任务
- **THEN** 系统更新父待办进度
- **AND** 重新计算完成百分比

#### Scenario: 删除子任务
- **WHEN** 用户删除子任务
- **THEN** 系统从数据库移除子任务
- **AND** 更新父待办进度

## MODIFIED Requirements

### Requirement: 待办进度
系统应跟踪并显示带子任务的待办进度，支持时间追踪。

#### Scenario: 计算进度
- **WHEN** 待办有子任务
- **THEN** 系统计算进度为：已完成子任务 / 总子任务
- **AND** 显示进度条或百分比

#### Scenario: 自动更新进度
- **WHEN** 子任务完成状态改变
- **THEN** 系统自动重新计算父待办进度
- **AND** 更新显示

#### Scenario: 时间追踪
- **WHEN** 用户开始工作于待办
- **THEN** 系统记录开始时间
- **AND** 用户可以记录工作时间
- **AND** 显示时间统计报告

## ADDED Requirements

### Requirement: 任务依赖
系统应支持任务之间的依赖关系。

#### Scenario: 设置任务依赖
- **WHEN** 用户编辑待办
- **THEN** 系统允许设置前置任务
- **AND** 依赖关系可视化
- **AND** 自动计算关键路径

#### Scenario: 依赖关系检查
- **WHEN** 用户尝试完成有未完成前置任务的任务
- **THEN** 系统提示依赖未完成
- **AND** 可选择强制完成

### Requirement: 任务模板
系统应支持任务模板功能。

#### Scenario: 创建任务模板
- **WHEN** 用户创建模板
- **THEN** 系统保存任务结构和字段
- **AND** 支持模板管理

#### Scenario: 从模板创建任务
- **WHEN** 用户选择从模板创建
- **THEN** 系统使用模板填充初始内容
- **AND** 用户可以修改字段

### Requirement: 批量操作
系统应支持待办的批量操作。

#### Scenario: 批量修改状态
- **WHEN** 用户选择多个待办
- **THEN** 系统允许批量修改状态
- **AND** 批量修改优先级
- **AND** 批量删除

### Requirement: 高级筛选和排序
系统应支持高级筛选和自定义排序。

#### Scenario: 多条件筛选
- **WHEN** 用户使用筛选功能
- **THEN** 系统支持多条件组合筛选
- **AND** 支持保存筛选条件

#### Scenario: 自定义排序
- **WHEN** 用户选择排序
- **THEN** 系统支持自定义排序规则
- **AND** 支持多字段排序
