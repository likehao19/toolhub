# 变更：优化笔记管理界面和功能

## 原因
当前笔记管理模块的界面布局不够合理，编辑器操作分散，编辑区域不够宽阔，文件列表信息不够全面，标签功能未充分利用，导出和批量导入功能位置不当。同时缺少对其他文件类型的支持（Excel、Word、TXT等），Markdown编辑器缺少常用格式化工具栏，文件管理缺少拖拽排序等功能。需要全面优化界面布局、增强编辑器功能、完善文件管理能力，使其达到现代化生产力工具的标准。

## 变更内容
- **MODIFIED**: 笔记管理界面布局 - 优化编辑器布局，操作栏置顶，编辑时隐藏文件夹列表，扩大编辑区域
- **MODIFIED**: 文件列表展示 - 增强文件列表信息展示（文件类型图标、文件大小、标签、修改时间等），添加排序筛选功能
- **MODIFIED**: 新建文件流程 - 改为内联输入框方式，无需弹窗，失焦自动保存
- **MODIFIED**: 文件夹图标 - 使用展开/折叠图标替代文件夹图标
- **MODIFIED**: 编辑器工具栏 - 统一编辑器操作到同一行，同类型操作使用下拉菜单，展示Markdown常用格式化功能（加粗、代码块等）
- **MODIFIED**: 标签功能 - 充分利用标签功能，在文件列表中显示标签，支持按标签筛选
- **MODIFIED**: 导出和批量导入 - 从编辑器内移至文件列表操作栏
- **MODIFIED**: 文章标题编辑 - 优化标题编辑体验
- **ADDED**: 多文件类型支持 - 支持编辑Excel、Word、TXT等文件类型，使用开源组件
- **ADDED**: 目录展示功能 - MD和Word文件支持目录展示，可点击跳转
- **ADDED**: 文件拖拽排序 - 支持文件和文件夹的拖拽排序，使用拖拽图标
- **ADDED**: 模板选择优化 - 新建文件时展示模板选择，不同类型文件有不同模板，支持模板CRUD
- **ADDED**: 资源文件管理 - MD中的图片和视频按随机ID保存到对应文件夹
- **ADDED**: Markdown实时预览 - 参考开源MD编辑器，支持Markdown语法同时展示图片，无需切换预览模式
- **MODIFIED**: 整体主题 - 采用白色主题，简约而不失功能全面

## 影响
- **受影响规范**: 
  - `notes` (修改)
- **受影响代码**:
  - `src/views/Notes.vue` - 需要重构界面布局和交互逻辑
  - `src/components/` - 可能需要新增编辑器组件、文件列表组件
  - `src/utils/notes.js` - 需要扩展文件类型支持
  - `src/utils/noteTemplate.js` - 需要扩展模板系统支持多文件类型
  - 数据库Schema - 可能需要扩展以支持新文件类型和模板
