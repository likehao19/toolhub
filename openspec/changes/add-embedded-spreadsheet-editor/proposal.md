# 变更：新增离线内嵌表格编辑器（XLSX/WPS 高兼容）

## 原因
当前 `.xlsx` 在应用内采用“基础网格表格”实现，无法与 WPS 的显示与编辑体验保持一致（样式、行列尺寸、公式、图片等）。用户需要在应用内离线完成创建、预览、编辑，并与 WPS 实现双向互操作。

## 变更内容
- **MODIFIED**: Notes 模块的 `.xlsx` 编辑/预览能力，从基础表格升级为离线内嵌电子表格引擎
- **ADDED**: `.xlsx` 的创建/打开/编辑/预览/保存全流程（以 `xlsx` 为唯一存储格式）
- **ADDED**: 与 WPS 的互操作保障与回归样例库（用例文件集 + 打开/保存一致性检查）
- **REMOVED**: 现有 Handsontable `.xlsx` 编辑路径（或仅保留为降级模式，具体见设计）

## 约束与现实边界
- **离线 + 开源 + 内嵌** 前提下，目标是“尽可能接近 WPS”，但无法承诺 100% UI 与行为完全一致（WPS/Office 存在大量专有实现与边界行为）。
- 若必须达到“肉眼无差别”，需要放宽约束（例如：商业表格控件或直接调用/嵌入 WPS/Office 引擎）。

## 影响
- **受影响规范**:
  - `notes`（修改）
- **受影响代码**:
  - Notes/NoteView 的 `.xlsx` 打开、预览、编辑、保存流程
  - 表格组件与适配层（新增）
  - 现有 Handsontable ExcelEditor（移除/降级）
