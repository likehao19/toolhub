## ADDED Requirements

### Requirement: 书签存储
系统应在 SQLite 数据库中存储书签及其元数据。

#### Scenario: 添加书签
- **WHEN** 用户点击"添加书签"
- **THEN** 系统打开表单，包含字段：标题、URL、描述、标签、分类
- **AND** 保存时，将书签存储到 SQLite 数据库
- **AND** 在列表中显示书签

#### Scenario: 查看书签列表
- **WHEN** 用户导航到书签
- **THEN** 系统显示所有存储的书签
- **AND** 显示标题、URL 预览和图标（如果可用）
- **AND** 支持网格或列表视图

#### Scenario: 在浏览器中打开书签
- **WHEN** 用户点击书签
- **THEN** 系统在默认浏览器中打开 URL
- **AND** 使用 Tauri opener 插件

#### Scenario: 编辑书签
- **WHEN** 用户在书签上点击"编辑"
- **THEN** 系统打开包含当前值的编辑表单
- **AND** 用户可以修改任何字段
- **AND** 更改保存到数据库

#### Scenario: 删除书签
- **WHEN** 用户在书签上点击"删除"
- **THEN** 系统提示确认
- **AND** 确认后从数据库移除书签

#### Scenario: 搜索书签
- **WHEN** 用户在搜索框中输入
- **THEN** 系统按标题、URL、标签或描述过滤书签
- **AND** 实时更新结果

### Requirement: 书签组织
系统应支持使用分类和标签组织书签。

#### Scenario: 书签分类
- **WHEN** 用户添加或编辑书签
- **THEN** 用户可以将其分配到分类
- **AND** 书签可以按分类过滤

#### Scenario: 书签标签
- **WHEN** 用户添加或编辑书签
- **THEN** 用户可以添加多个标签
- **AND** 书签可以按标签过滤

#### Scenario: 按分类查看
- **WHEN** 用户从侧边栏选择分类
- **THEN** 系统仅显示该分类中的书签

### Requirement: 书签图标
系统应尝试获取并显示书签的网站图标。

#### Scenario: 显示图标
- **WHEN** 书签被添加或加载
- **THEN** 系统尝试从网站获取图标
- **AND** 在书签标题旁显示图标
- **AND** 如果不可用则回退到默认图标
