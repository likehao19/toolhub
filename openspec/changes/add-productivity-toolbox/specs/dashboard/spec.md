## ADDED Requirements

### Requirement: 首页仪表板
系统应提供一个首页仪表板，展示最近活动和所有模块的快速访问。

#### Scenario: 应用启动时查看仪表板
- **WHEN** 用户打开应用
- **THEN** 默认显示仪表板
- **AND** 展示最近笔记、密码、书签、今日待办和今日日程

#### Scenario: 查看最近笔记
- **WHEN** 用户在仪表板上
- **THEN** 系统显示最近修改的笔记列表
- **AND** 每个笔记显示标题、修改时间和预览

#### Scenario: 查看最近密码
- **WHEN** 用户在仪表板上
- **THEN** 系统显示最近访问的密码
- **AND** 密码以掩码形式显示以确保安全

#### Scenario: 查看常用书签
- **WHEN** 用户在仪表板上
- **THEN** 系统显示常用书签
- **AND** 书签可点击在浏览器中打开

#### Scenario: 查看今日待办
- **WHEN** 用户在仪表板上
- **THEN** 系统显示今日计划的待办事项
- **AND** 显示完成状态和进度

#### Scenario: 查看今日日程
- **WHEN** 用户在仪表板上
- **THEN** 系统显示今日的日程事件
- **AND** 显示事件时间和描述

### Requirement: 命令输入
系统应提供一个命令输入框，允许用户使用 `@` 符号命令快速创建内容。

#### Scenario: 通过命令创建笔记
- **WHEN** 用户在命令输入框中输入 `@note`
- **THEN** 系统打开笔记创建对话框
- **AND** 聚焦到笔记编辑器

#### Scenario: 通过命令创建待办
- **WHEN** 用户在命令输入框中输入 `@todo`
- **THEN** 系统打开待办创建对话框

#### Scenario: 通过命令创建日程事件
- **WHEN** 用户在命令输入框中输入 `@calendar` 或 `@event`
- **THEN** 系统打开日程事件创建对话框

#### Scenario: 命令自动补全
- **WHEN** 用户在命令输入框中输入 `@`
- **THEN** 系统显示可用命令列表
- **AND** 用户可以从列表中选择或继续输入
