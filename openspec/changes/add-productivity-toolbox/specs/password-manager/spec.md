## ADDED Requirements

### Requirement: 密码存储
系统应在 SQLite 数据库中安全地存储密码，并进行加密。

#### Scenario: 创建密码条目
- **WHEN** 用户点击"添加密码"
- **THEN** 系统打开表单，包含字段：标题、用户名、密码、网站、备注
- **AND** 保存时，将加密的密码存储到 SQLite 数据库
- **AND** 在密码列表中显示条目

#### Scenario: 查看密码列表
- **WHEN** 用户导航到密码管理器
- **THEN** 系统显示所有存储的密码
- **AND** 密码默认以掩码形式显示
- **AND** 显示标题、用户名和网站

#### Scenario: 查看密码详情
- **WHEN** 用户点击密码条目
- **THEN** 系统显示完整详情
- **AND** 可以通过"显示密码"按钮显示密码
- **AND** 支持复制到剪贴板

#### Scenario: 编辑密码条目
- **WHEN** 用户在密码条目上点击"编辑"
- **THEN** 系统打开包含当前值的编辑表单
- **AND** 用户可以修改任何字段
- **AND** 更改保存到数据库

#### Scenario: 删除密码条目
- **WHEN** 用户在密码条目上点击"删除"
- **THEN** 系统提示确认
- **AND** 确认后从数据库移除条目

#### Scenario: 搜索密码
- **WHEN** 用户在搜索框中输入
- **THEN** 系统按标题、用户名或网站过滤密码
- **AND** 实时更新结果

### Requirement: 密码安全
系统应加密密码并提供安全的访问控制。

#### Scenario: 保存时加密密码
- **WHEN** 用户保存密码
- **THEN** 系统在存储前加密密码值
- **AND** 使用安全的加密算法
- **AND** 解密密钥安全管理

#### Scenario: 列表中掩码密码
- **WHEN** 密码在列表中显示
- **THEN** 密码值以 `••••••` 掩码显示
- **AND** 仅在用户明确请求时显示

### Requirement: 密码生成器
系统应提供密码生成器用于创建安全密码。

#### Scenario: 生成密码
- **WHEN** 用户点击"生成密码"
- **THEN** 系统生成随机安全密码
- **AND** 用户可以配置长度和字符类型
- **AND** 生成的密码可以复制到剪贴板
