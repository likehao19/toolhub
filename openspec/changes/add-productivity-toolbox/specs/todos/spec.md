## ADDED Requirements

### Requirement: 待办管理
系统应提供支持子任务和进度跟踪的待办管理。

#### Scenario: 创建待办
- **WHEN** 用户点击"新建待办"或使用 `@todo` 命令
- **THEN** 系统打开表单，包含字段：标题、描述、截止日期、优先级、分类
- **AND** 保存时，将待办存储到 SQLite 数据库
- **AND** 在列表中显示待办

#### Scenario: 查看待办列表
- **WHEN** 用户导航到待办
- **THEN** 系统显示所有待办
- **AND** 显示标题、截止日期、优先级和完成状态
- **AND** 支持按状态、日期、优先级过滤

#### Scenario: 完成待办
- **WHEN** 用户勾选完成复选框
- **THEN** 系统将待办标记为已完成
- **AND** 更新进度百分比
- **AND** 移动到已完成部分（如果过滤）

#### Scenario: 编辑待办
- **WHEN** 用户在待办上点击"编辑"
- **THEN** 系统打开包含当前值的编辑表单
- **AND** 用户可以修改任何字段
- **AND** 更改保存到数据库

#### Scenario: 删除待办
- **WHEN** 用户在待办上点击"删除"
- **THEN** 系统提示确认
- **AND** 确认后从数据库移除待办

### Requirement: 子任务
系统应支持为待办创建子任务。

#### Scenario: 添加子任务
- **WHEN** 用户在待办上点击"添加子任务"
- **THEN** 系统创建新子任务
- **AND** 子任务与父待办关联
- **AND** 在父项下缩进显示

#### Scenario: 完成子任务
- **WHEN** 用户完成子任务
- **THEN** 系统更新父待办进度
- **AND** 重新计算完成百分比

#### Scenario: 删除子任务
- **WHEN** 用户删除子任务
- **THEN** 系统从数据库移除子任务
- **AND** 更新父待办进度

### Requirement: 待办进度
系统应跟踪并显示带子任务的待办进度。

#### Scenario: 计算进度
- **WHEN** 待办有子任务
- **THEN** 系统计算进度为：已完成子任务 / 总子任务
- **AND** 显示进度条或百分比

#### Scenario: 自动更新进度
- **WHEN** 子任务完成状态改变
- **THEN** 系统自动重新计算父待办进度
- **AND** 更新显示

### Requirement: 今日待办
系统应在仪表板上显示今日待办。

#### Scenario: 显示今日待办
- **WHEN** 用户查看仪表板
- **THEN** 系统显示截止日期为今日的待办
- **AND** 显示完成状态
- **AND** 高亮显示过期待办

#### Scenario: 按日期过滤
- **WHEN** 用户导航到待办页面
- **THEN** 用户可以按"今日"、"本周"、"过期"或"全部"过滤
- **AND** 列表相应更新

### Requirement: 待办通知
系统应在待办截止时间到达时在桌面右下角弹出系统通知。

#### Scenario: 待办时间到达通知
- **WHEN** 待办的截止时间到达
- **THEN** 系统在桌面右下角弹出系统通知
- **AND** 通知显示待办标题和描述
- **AND** 通知显示截止时间信息
- **AND** 使用 Tauri 通知插件

#### Scenario: 待办即将到期通知
- **WHEN** 待办在指定时间前（如 15 分钟前）
- **THEN** 系统可以提前发送提醒通知
- **AND** 用户可以在设置中配置提前提醒时间

#### Scenario: 通知点击处理
- **WHEN** 用户点击待办通知
- **THEN** 系统打开应用并导航到待办详情页面
- **AND** 高亮显示该待办项

#### Scenario: 通知权限检查
- **WHEN** 系统尝试发送通知
- **THEN** 系统检查通知权限
- **AND** 如果未授权，提示用户授予通知权限
