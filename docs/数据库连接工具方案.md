# 数据库连接工具方案

## 概述

在工具箱中新增一个通用数据库连接管理工具，支持主流关系型与 NoSQL 数据库的连接、SQL/命令执行、结果浏览与导出。定位为轻量级桌面数据库客户端，类似 DBeaver / DataGrip 的核心子集。

## 难度评估

| 维度 | 难度 | 说明 |
|------|------|------|
| 前端 UI | ⭐⭐ 中等 | 连接管理 + SQL 编辑器 + 结果表格，可复用 Redis 客户端模式 |
| Rust 后端 | ⭐⭐⭐ 较高 | 需集成多种数据库 driver，统一连接池与结果序列化 |
| SQL 编辑器 | ⭐⭐ 中等 | CodeMirror/Monaco 集成，语法高亮 + 自动补全 |
| 整体工作量 | ⭐⭐⭐ 较高 | 预计 8-12 个文件，Rust 侧代码量最大 |

## 支持的数据库

### 关系型数据库

| 数据库 | Rust Crate | 默认端口 | 说明 |
|--------|-----------|---------|------|
| MySQL / MariaDB | `sqlx` (mysql) | 3306 | 最常用的开源关系型数据库 |
| PostgreSQL | `sqlx` (postgres) | 5432 | 功能最丰富的开源数据库 |
| SQLite | `sqlx` (sqlite) | — | 本地文件数据库，无需服务端 |
| SQL Server (MSSQL) | `tiberius` | 1433 | 微软商用数据库 |
| Oracle | `oracle` (via OCI) | 1521 | 企业级商用数据库（可选，依赖较重） |

### NoSQL 数据库

| 数据库 | Rust Crate | 默认端口 | 说明 |
|--------|-----------|---------|------|
| Redis | `redis` (已集成) | 6379 | 已有独立 Redis 客户端，此处可复用 |
| MongoDB | `mongodb` | 27017 | 文档型数据库 |

### 第一阶段优先实现

1. **MySQL** — 使用最广泛
2. **PostgreSQL** — 开发者高频使用
3. **SQLite** — 本地开发常用，无需服务端
4. **MongoDB** — NoSQL 代表

后续阶段再扩展 MSSQL、Oracle 等。

### 核心设计：统一抽象层

```rust
// Rust 伪代码 — DatabaseDriver trait
#[async_trait]
trait DatabaseDriver: Send + Sync {
    /// 测试连接
    async fn test_connection(&self) -> Result<(), String>;
    /// 执行查询，返回行列结果
    async fn execute_query(&self, sql: &str) -> Result<QueryResult, String>;
    /// 执行语句（INSERT/UPDATE/DELETE），返回影响行数
    async fn execute_statement(&self, sql: &str) -> Result<u64, String>;
    /// 获取数据库列表
    async fn list_databases(&self) -> Result<Vec<String>, String>;
    /// 获取表列表
    async fn list_tables(&self, database: &str) -> Result<Vec<TableInfo>, String>;
    /// 获取表结构（列信息）
    async fn describe_table(&self, database: &str, table: &str) -> Result<Vec<ColumnInfo>, String>;
    /// 获取表索引
    async fn list_indexes(&self, database: &str, table: &str) -> Result<Vec<IndexInfo>, String>;
    /// 取消正在执行的查询
    async fn cancel(&self) -> Result<(), String>;
}

#[derive(Serialize)]
struct QueryResult {
    columns: Vec<ColumnMeta>,   // 列名 + 类型
    rows: Vec<Vec<serde_json::Value>>,  // 行数据
    affected_rows: u64,
    execution_time_ms: u64,     // 执行耗时
}
```

### ConnConfig 结构

```rust
#[derive(Deserialize)]
struct ConnConfig {
    name: String,
    db_type: DbType,            // mysql | postgres | sqlite | mongodb
    host: Option<String>,       // SQLite 不需要
    port: Option<u16>,
    username: Option<String>,
    password: Option<String>,
    database: Option<String>,   // 默认数据库
    file_path: Option<String>,  // SQLite 文件路径
    // SSH 隧道
    ssh_enabled: bool,
    ssh_host: Option<String>,
    ssh_port: Option<u16>,
    ssh_user: Option<String>,
    ssh_key_path: Option<String>,
    // TLS/SSL
    ssl_enabled: bool,
    ssl_ca_path: Option<String>,
    // 连接参数
    connection_timeout: Option<u64>,  // 秒
    max_pool_size: Option<u32>,       // 连接池大小，默认 5
}
```

### 数据库表（SQLite 存储连接配置）

```sql
-- migration vN
CREATE TABLE IF NOT EXISTS db_connections (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    db_type TEXT NOT NULL,               -- 'mysql' | 'postgres' | 'sqlite' | 'mongodb' | 'mssql'
    host TEXT DEFAULT '127.0.0.1',
    port INTEGER,
    username TEXT DEFAULT '',
    password TEXT DEFAULT '',             -- 加密存储
    default_database TEXT DEFAULT '',
    file_path TEXT DEFAULT '',            -- SQLite 专用
    ssh_enabled INTEGER DEFAULT 0,
    ssh_host TEXT DEFAULT '',
    ssh_port INTEGER DEFAULT 22,
    ssh_user TEXT DEFAULT '',
    ssh_key_path TEXT DEFAULT '',
    ssl_enabled INTEGER DEFAULT 0,
    ssl_ca_path TEXT DEFAULT '',
    connection_timeout INTEGER DEFAULT 10,
    max_pool_size INTEGER DEFAULT 5,
    color TEXT DEFAULT '',                -- 连接标识色
    sort_order INTEGER DEFAULT 0,
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    last_used_at TEXT,
    -- 保存用户最近打开的 tab / 查询
    last_state_json TEXT DEFAULT '{}'
);

-- 保存用户的 SQL 查询历史
CREATE TABLE IF NOT EXISTS db_query_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    connection_id INTEGER NOT NULL,
    sql_text TEXT NOT NULL,
    execution_time_ms INTEGER,
    row_count INTEGER,
    is_favorite INTEGER DEFAULT 0,        -- 收藏的查询
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    FOREIGN KEY (connection_id) REFERENCES db_connections(id) ON DELETE CASCADE
);

-- 保存用户的 SQL 片段 (snippets)
CREATE TABLE IF NOT EXISTS db_saved_queries (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    connection_id INTEGER,                -- NULL = 全局
    name TEXT NOT NULL,
    sql_text TEXT NOT NULL,
    description TEXT DEFAULT '',
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

## 功能模块

### 1. 连接管理

- 左侧面板：连接列表，按数据库类型分组显示
- 每个连接显示：名称、类型图标、host:port、颜色标识、在线状态
- 操作：新建 / 编辑 / 复制 / 删除 / 测试连接
- 连接表单根据 db_type 动态显示不同字段（如 SQLite 只需文件路径）
- 支持 SSH 隧道配置（折叠面板）
- 支持 SSL/TLS 配置
- 连接分组与拖拽排序

### 2. 数据库导航树

```
📦 My MySQL Server
├── 📁 database_1
│   ├── 📋 users (52,341 rows)
│   ├── 📋 orders (128,903 rows)
│   └── 📋 products (1,205 rows)
├── 📁 database_2
│   └── ...
└── 📁 information_schema
```

- 树形结构：连接 → 数据库 → 表/视图/函数
- 右键菜单：查看数据、查看结构、复制表名、导出、删除
- 表名搜索过滤
- 懒加载（展开时才请求子节点）

### 3. SQL 编辑器

- **编辑器**：集成 CodeMirror 6（已有代码格式化工具经验）
- SQL 语法高亮（根据数据库类型切换方言）
- 基础自动补全：表名、列名、SQL 关键字
- 多 Tab 支持：每个 Tab 一个查询编辑器
- 执行快捷键：`Ctrl+Enter` 执行全部，`Ctrl+Shift+Enter` 执行选中
- 执行状态指示：运行中 / 成功 / 失败
- 查询耗时显示
- 查询历史（自动保存最近 100 条）
- SQL 片段保存与管理

### 4. 结果展示

- **表格模式**：虚拟滚动表格（支持大数据量），列可排序、调宽
- **JSON 模式**：树形 JSON 查看器（MongoDB 结果天然适合）
- **统计栏**：行数、列数、执行耗时、影响行数
- 单元格操作：复制值、查看完整内容（对长文本/JSON）
- 导出功能：CSV / JSON / SQL INSERT / Excel
- 分页浏览（默认 500 行一页）

### 5. 表结构查看

- **列信息**：列名、类型、是否可空、默认值、注释
- **索引信息**：索引名、类型、列
- **DDL 查看**：CREATE TABLE 语句（可复制）
- **关系图**：外键关系可视化（后续阶段）

### 6. 表数据编辑（后续阶段）

- 行内编辑：双击单元格直接修改
- 新增行 / 删除行
- 修改预览（显示将要执行的 SQL）
- 提交 / 回滚

### 7. MongoDB 专属功能

- 集合浏览（替代表）
- 文档 JSON 编辑器
- 查询使用 MongoDB 查询语法（JSON filter）而非 SQL
- Aggregation Pipeline 构建器（后续阶段）

## 风险与注意事项

1. **编译体积**：`sqlx` 多数据库 feature 会增加约 5-10MB 二进制体积
2. **Oracle 驱动**：需要 OCI 客户端库，Windows 下配置复杂，建议放最后或不支持
3. **SSH 隧道**：Rust 端需 `russh` 或 `ssh2` crate 建立隧道，复杂度较高
4. **MongoDB 查询语法**：与 SQL 完全不同，前端需要切换编辑器模式
5. **大结果集**：前端需虚拟滚动，避免一次渲染数万行 DOM
6. **密码安全**：连接密码需加密存储，不能明文存 SQLite
7. **连接池管理**：需要处理连接超时、断线重连、最大连接数限制
