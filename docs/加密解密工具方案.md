# 加密解密工具方案

## 概述

在工具箱中新增一个全能型加密解密工具，集成哈希摘要、对称加密、非对称加密、编码转换等 30+ 种算法于一体。纯前端实现，无需 Rust 后端，定位为开发者日常加解密的瑞士军刀。

## 技术方案

### 架构

```
┌──────────────────────────────────────────────────────┐
│  Vue 前端（Element Plus）                              │
│  ┌──────────┬───────────┬──────────┬──────────┐      │
│  │ 分类导航  │ 算法选择   │ 输入输出  │ 说明面板  │      │
│  └──────────┴───────────┴──────────┴──────────┘      │
│              ↕ crypto-js / jsencrypt                   │
├──────────────────────────────────────────────────────┤
│  纯 JS 加密库（无 Rust 依赖）                           │
│  ┌────────────────┬─────────────────┐                │
│  │  crypto-js     │  jsencrypt(RSA) │                │
│  └────────────────┴─────────────────┘                │
└──────────────────────────────────────────────────────┘
```

- **前端**：Vue 3 + Element Plus，三栏布局（分类导航 + 主操作区 + 说明面板）
- **加密库**：`crypto-js`（哈希 + 对称加密）+ `jsencrypt`（RSA）
- **无需 Rust 后端**：所有计算在浏览器完成

### 前端依赖

```json
{
  "crypto-js": "^4.2.0",
  "jsencrypt": "^3.3.2"
}
```

## 支持算法一览（30+ 种）

### 1. 哈希摘要（12 种）

| 算法 | 输出长度 | 说明 |
|------|---------|------|
| MD5 | 128 bit | 经典摘要，用于文件校验 |
| SHA-1 | 160 bit | 已不推荐 |
| SHA-224 | 224 bit | SHA-2 家族 |
| SHA-256 | 256 bit | 最常用，区块链/TLS |
| SHA-384 | 384 bit | SHA-2 家族 |
| SHA-512 | 512 bit | 最强 SHA-2 |
| SHA3-256 | 256 bit | 第三代哈希标准 |
| SHA3-512 | 512 bit | SHA-3 家族 |
| RIPEMD-160 | 160 bit | 比特币地址使用 |
| HMAC-MD5 | 128 bit | 带密钥消息认证码 |
| HMAC-SHA256 | 256 bit | API 签名常用 |
| HMAC-SHA512 | 512 bit | 高强度 HMAC |

### 2. 对称加密（5 种）

| 算法 | 密钥长度 | 说明 |
|------|---------|------|
| AES | 128/192/256 bit | 当今最广泛使用的对称加密 |
| DES | 56 bit | 已过时，仅作学习 |
| 3DES (Triple DES) | 168 bit | DES 增强版 |
| Rabbit | 128 bit | 高性能流密码 |
| RC4 | 可变 | 已弃用的流密码 |

**对称加密参数支持：**
- 加密模式：CBC / ECB / CFB / OFB / CTR
- 填充方式：Pkcs7 / ZeroPadding / NoPadding
- 密钥长度选择：128 / 192 / 256 bit
- 输出格式：Hex / Base64

### 3. 非对称加密（1 种）

| 算法 | 密钥长度 | 说明 |
|------|---------|------|
| RSA | 1024/2048/4096 bit | 公钥加密、私钥解密 |

**RSA 功能：**
- 公钥加密 / 私钥解密
- 一键生成 2048 bit 密钥对
- PEM 格式密钥输入

### 4. 编码转换（11 种）

| 编码 | 说明 |
|------|------|
| Base64 | 二进制→ASCII，支持 UTF-8 |
| Base32 | A-Z + 2-7 编码 |
| Hex | 十六进制编码 |
| URL Encode | 百分号编码（RFC 3986） |
| HTML Encode | HTML 实体转义 |
| Unicode | \uXXXX 转义 |
| UTF-8 Hex | 查看 UTF-8 字节序列 |
| ASCII | 字符↔ASCII 码值 |
| Binary | 二进制 01 表示 |
| Octal | 八进制编码 |
| Morse | 摩尔斯电码 |

### 5. 其他工具（3 种）

| 工具 | 说明 |
|------|------|
| JWT Decode | 解析 JWT Token 的 Header + Payload |
| Bcrypt Gen | 生成 Bcrypt 风格哈希（简化版） |
| CRC32 | 循环冗余校验 |

## 特点与优势

1. **种类丰富**：30+ 种算法，覆盖开发中几乎所有加解密需求
2. **纯前端**：无需 Rust 后端，crypto-js + jsencrypt 即可完成
3. **单文件实现**：一个 Vue 组件包含全部逻辑，维护简单
4. **参数灵活**：对称加密支持多种模式、填充、密钥长度、输出格式
5. **右侧说明面板**：每个算法附带说明、属性信息，方便学习
6. **编码双向**：编码类工具支持编码/解码双向操作
7. **交换功能**：一键交换输入输出，方便反向操作

## 开发难度

**低**。全部为纯前端实现，crypto-js 提供了绝大部分算法，jsencrypt 处理 RSA。无 Rust 代码，无数据库表，无新增 Tauri command。核心工作量在 UI 布局和各算法的参数适配。
