# 代理工具方案（Proxy Manager）

> 集成 mihomo（Clash Meta）内核，实现类 Clash for Windows 的代理管理工具。

## 一、架构设计

```
┌──────────────────────────────────────────────────────────┐
│                    Tauri 应用（GUI 壳）                    │
│                                                          │
│  ┌──────────────────┐    ┌─────────────────────────────┐ │
│  │    Vue 3 前端     │    │        Rust 后端             │ │
│  │                  │    │                             │ │
│  │  订阅管理 UI     │◄──►│  mihomo 进程管理             │ │
│  │  节点选择/测速    │    │  ├ 启动 / 停止 / 重启        │ │
│  │  分流规则编辑     │    │  ├ 配置文件生成 (YAML)       │ │
│  │  流量实时监控     │    │  ├ 订阅下载 & 解析           │ │
```

### 核心原则

- **不造轮子**：代理协议（SS/VMess/Trojan/Hysteria）全部由 mihomo 内核处理
- **GUI 壳模式**：Tauri 应用只负责 UI + 进程管理 + 配置生成
- **REST API 通信**：通过 mihomo 的 `127.0.0.1:9090` RESTful API 控制一切
- **Sidecar 打包**：mihomo 二进制随应用分发，用户无需单独安装

## 二、功能模块

| 模块 | 功能 | 技术方案 |
|------|------|---------|
| **内核管理** | 启动/停止/重启 mihomo、状态监控、版本检测 | Tauri sidecar + 进程管理 |
| **订阅管理** | 导入/更新/删除订阅链接，自动解析节点 | Rust reqwest 下载 + Base64/YAML 解析 |
| **节点选择** | 代理组切换、节点延迟测速、自动选择 | mihomo API: `/proxies` |
| **分流规则** | 规则模式切换（全局/规则/直连）、自定义规则 | mihomo API: `/configs` + YAML 生成 |
| **流量监控** | 实时上传/下载速度、累计流量 | mihomo WebSocket: `/traffic` |
| **连接日志** | 实时连接列表、关闭连接、搜索过滤 | mihomo WebSocket: `/connections` |
| **系统代理** | 一键开关系统代理、开机自启代理 | Windows 注册表操作 |
| **设置** | 代理端口、API 端口、DNS 配置、TUN 模式 | YAML 配置文件生成 |

```rust
use tauri_plugin_shell::ShellExt;

#[tauri::command]
async fn start_proxy(app: tauri::AppHandle) -> Result<String, String> {
    let config_path = get_config_path(&app);

    let (mut rx, child) = app.shell()
        .sidecar("mihomo")
        .map_err(|e| e.to_string())?
        .args([
            "-d", &config_dir,        // 配置目录
            "-ext-ctl", "127.0.0.1:9090",  // API 地址
        ])
        .spawn()
        .map_err(|e| e.to_string())?;

    // 保存 child 进程引用，用于后续停止
    // 监听 stdout/stderr 推送日志到前端

    Ok("Proxy started".into())
}

#[tauri::command]
async fn stop_proxy() -> Result<(), String> {
    // kill child process
}
```

### 4.4 系统代理设置（Windows）

**通过注册表设置 IE/系统代理：**

```rust
use winreg::RegKey;
use winreg::enums::*;

const PROXY_REG_PATH: &str =
    r"Software\Microsoft\Windows\CurrentVersion\Internet Settings";

#[tauri::command]
fn enable_system_proxy(port: u16) -> Result<(), String> {
    let hkcu = RegKey::predef(HKEY_CURRENT_USER);
    let key = hkcu.open_subkey_with_flags(PROXY_REG_PATH, KEY_WRITE)
        .map_err(|e| e.to_string())?;

    // 开启代理
    key.set_value("ProxyEnable", &1u32)
        .map_err(|e| e.to_string())?;

    // 设置代理地址
    key.set_value("ProxyServer", &format!("127.0.0.1:{}", port))
        .map_err(|e| e.to_string())?;

    // 设置绕过列表
    key.set_value("ProxyOverride", &"localhost;127.*;10.*;172.16.*;<local>")
        .map_err(|e| e.to_string())?;

    // 通知系统设置已更改
    notify_proxy_change();
    Ok(())
}

#[tauri::command]
fn disable_system_proxy() -> Result<(), String> {
    let hkcu = RegKey::predef(HKEY_CURRENT_USER);
    let key = hkcu.open_subkey_with_flags(PROXY_REG_PATH, KEY_WRITE)
        .map_err(|e| e.to_string())?;

    key.set_value("ProxyEnable", &0u32)
        .map_err(|e| e.to_string())?;

    notify_proxy_change();
    Ok(())
}

// 通知系统代理设置已更改（让浏览器等应用立即生效）
fn notify_proxy_change() {
    use windows::Win32::Networking::WinInet::*;
    unsafe {
        InternetSetOptionW(None, INTERNET_OPTION_SETTINGS_CHANGED, None, 0);
        InternetSetOptionW(None, INTERNET_OPTION_REFRESH, None, 0);
    }
}
```

### 4.5 订阅解析

**Rust 端下载并解析订阅：**

```rust
#[tauri::command]
async fn import_subscription(url: String, name: String) -> Result<SubInfo, String> {
    let client = reqwest::Client::builder()
        .user_agent("clash")    // 有些订阅需要特定 UA
        .build().map_err(|e| e.to_string())?;

    let resp = client.get(&url).send().await.map_err(|e| e.to_string())?;

    // 提取订阅信息（流量/到期时间）
    let sub_info = resp.headers()
        .get("subscription-userinfo")
        .and_then(|v| v.to_str().ok())
        .map(parse_sub_userinfo);

    let content = resp.text().await.map_err(|e| e.to_string())?;

    // 尝试 YAML 解析
    if let Ok(config) = serde_yaml::from_str::<ClashConfig>(&content) {
        return Ok(SubInfo {
            name, proxies: config.proxies, info: sub_info
        });
    }

    // 尝试 Base64 解码后再解析
    if let Ok(decoded) = base64_decode(&content) {
        let lines: Vec<&str> = decoded.lines().collect();
        let proxies = lines.iter()
            .filter_map(|line| parse_proxy_uri(line))
            .collect();
        return Ok(SubInfo { name, proxies, info: sub_info });
    }

    Err("无法解析订阅内容".into())
}
```

### 4.6 配置文件生成

**根据用户设置动态生成 config.yaml：**

```rust
fn generate_config(settings: &ProxySettings, subscriptions: &[SubInfo]) -> String {
    let mut config = format!(r#"
mixed-port: {port}
allow-lan: {allow_lan}
mode: {mode}
log-level: info
external-controller: 127.0.0.1:{api_port}
secret: '{secret}'

dns:
  enable: true
  enhanced-mode: {dns_mode}
  nameserver:
    - {dns_default}
  fallback:
    - {dns_fallback}

"#,
        port = settings.mixed_port,
        allow_lan = settings.allow_lan,
        mode = settings.mode,
        api_port = settings.api_port,
        secret = settings.secret,
        dns_mode = settings.dns_mode,
        dns_default = settings.dns_default,
        dns_fallback = settings.dns_fallback,
    );

    // 写入所有订阅的节点
    config.push_str("proxies:\n");
    for sub in subscriptions {
        for proxy in &sub.proxies {
            config.push_str(&format!("  - {}\n", serde_yaml::to_string(proxy).unwrap()));
        }
    }

    // 写入代理组
    config.push_str(&generate_proxy_groups(subscriptions));

    // 写入规则
    config.push_str(&generate_rules(settings));

    config
}
```

```
src-tauri/
  binaries/
    mihomo-x86_64-pc-windows-msvc.exe    ← mihomo 内核二进制
  src/
    commands/
      proxy.rs                            ← 进程管理 + API 客户端
      proxy_config.rs                     ← 配置生成 + 订阅解析
      proxy_system.rs                     ← 系统代理设置

src/
  views/
    ProxyManager.vue                      ← 主视图（左侧导航 + 右侧面板）
  utils/
    proxyManager.js                       ← localStorage + WebSocket 封装
```

### 4.9 存储

| 数据 | 存储方式 | Key |
|------|---------|-----|
| 订阅列表 | localStorage | `proxy-subscriptions` |
| 代理设置 | localStorage | `proxy-settings` |
| 上次选中的节点 | localStorage | `proxy-selected` |
| mihomo 配置 | 文件系统 | `{appData}/proxy/config.yaml` |
| mihomo 内核 | sidecar | `binaries/mihomo` |

## 五、mihomo API 速查

| 端点 | 方法 | 说明 |
|------|------|------|
| `/proxies` | GET | 获取所有代理组和节点 |
| `/proxies/{name}` | GET | 获取指定代理详情 |
| `/proxies/{name}` | PUT | 切换代理组中的节点 `{"name":"节点名"}` |
| `/proxies/{name}/delay` | GET | 测试节点延迟 `?timeout=5000&url=...` |
| `/configs` | GET | 获取当前配置 |
| `/configs` | PATCH | 更新配置 `{"mode":"rule"}` |
| `/configs` | PUT | 重载配置文件 `{"path":"config.yaml"}` |
| `/rules` | GET | 获取所有规则 |
| `/connections` | GET/WS | 获取/实时监控连接 |
| `/connections` | DELETE | 关闭所有连接 |
| `/connections/{id}` | DELETE | 关闭指定连接 |
| `/traffic` | WS | 实时流量 `{"up":123,"down":456}` 字节/秒 |
| `/providers/proxies` | GET | 获取代理提供者 |
| `/providers/proxies/{name}` | PUT | 更新代理提供者 |
| `/dns/query` | GET | DNS 查询 `?name=example.com&type=A` |

## 六、实现优先级

### P0 — 核心可用
1. mihomo sidecar 打包 + 进程启停
2. 配置文件生成（端口/模式/DNS）
3. 订阅导入（URL 下载 + YAML/Base64 解析）
4. 节点列表展示 + 切换代理
5. 系统代理开关（注册表）
6. 概览面板（状态 + 实时流量）

### P1 — 完善体验
7. 节点延迟测速（单个/批量）
8. 代理模式切换（规则/全局/直连）
9. 连接日志（实时列表 + 关闭连接）
10. 订阅自动更新 + 流量信息展示
11. 分流规则编辑（自定义域名/IP 规则）

### P2 — 高级功能
12. TUN 模式（需管理员权限 + wintun 驱动）
13. DNS 高级设置（fake-ip / redir-host）
14. 规则提供者（远程规则集）
15. 代理链（链式代理）
16. mihomo 内核自动更新

## 七、注意事项

1. **mihomo 二进制获取**：从 [mihomo releases](https://github.com/MetaCubeX/mihomo/releases) 下载对应平台版本
2. **安全**：API secret 随机生成，仅本地通信
3. **退出清理**：应用退出时必须 kill mihomo 进程 + 关闭系统代理
4. **权限**：系统代理设置写 HKCU（用户级），无需管理员权限；TUN 模式需管理员
5. **防火墙**：首次启动可能触发 Windows 防火墙提示
