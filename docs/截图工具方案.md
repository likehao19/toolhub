# 截图工具方案（Screenshot Tool）

> 类 Snipaste 的截图 + 贴图 + 标注工具，集成到工具箱中。

## 一、功能全景

```

│                   Screenshot Tool                        │
```

## 四、技术架构

### 4.1 核心技术选型

| 层面 | 方案 | 说明 |
|------|------|------|
| **屏幕捕获** | Rust `xcap` crate | 跨平台屏幕截图，支持显示器/窗口枚举 |
| **透明覆盖窗口** | Tauri 透明窗口 | `transparent: true` + `decorations: false` |
| **区域选择** | Canvas 绘制 | 全屏 Canvas 上绘制选区框 + 遮罩 |
| **标注绘制** | Canvas 2D API | 原生 Canvas 绑定鼠标事件进行图形绘制 |
| **贴图窗口** | Tauri 动态创建窗口 | 每个贴图一个独立无边框窗口 |
| **全局快捷键** | `tauri-plugin-global-shortcut` | 已集成，注册 F1 等热键 |
| **剪贴板** | `tauri-plugin-clipboard-manager` | 已集成，写入图片到剪贴板 |
| **文件保存** | `tauri-plugin-dialog` + `tauri-plugin-fs` | 已集成，另存为对话框 |
| **取色** | Canvas `getImageData` | 从截图数据中读取像素颜色值 |

### 4.2 截图流程技术细节

```
Step 1: 全局热键触发
  └→ tauri-plugin-global-shortcut 监听 F1

Step 2: Rust 后端捕获全屏
  └→ xcap::Monitor::all() → 获取所有显示器
  └→ monitor.capture_image() → 捕获全屏图像
  └→ 编码为 base64 PNG 传给前端

Step 3: 创建截图覆盖窗口
  └→ Tauri WebviewWindow::builder()
      .transparent(true)         // 透明
      .decorations(false)        // 无标题栏
      .fullscreen(true)          // 全屏
      .always_on_top(true)       // 置顶
      .skip_taskbar(true)        // 不显示在任务栏

Step 4: 前端渲染截图覆盖层
  └→ <canvas> 绘制全屏截图 + 半透明遮罩
  └→ 鼠标事件处理选区框
  └→ 放大镜：canvas.getImageData() 读取鼠标附近像素

Step 5: 标注编辑
  └→ 在选区 canvas 上叠加标注层
  └→ 操作历史栈（撤销/重做）
  └→ 各标注工具绑定不同的 canvas 绑定事件

Step 6: 输出
  └→ canvas.toBlob() → 合并标注层
  └→ 复制: clipboard.writeImage()
  └→ 保存: dialog.save() + fs.writeFile()
  └→ 贴图: 创建新 Tauri 窗口显示图片
```

### 4.3 Rust 后端命令

```rust
// src-tauri/src/commands/screenshot.rs

/// 捕获全部显示器截图
#[tauri::command]
async fn capture_all_screens() -> Result<Vec<ScreenCapture>, String> {
    let monitors = xcap::Monitor::all().map_err(|e| e.to_string())?;
    let mut captures = Vec::new();

    for monitor in monitors {
        let image = monitor.capture_image().map_err(|e| e.to_string())?;
        let mut buf = Vec::new();
        image.write_to(&mut std::io::Cursor::new(&mut buf),
            image::ImageFormat::Png).map_err(|e| e.to_string())?;

        captures.push(ScreenCapture {
            id: monitor.id(),
            name: monitor.name().to_string(),
            x: monitor.x(),
            y: monitor.y(),
            width: monitor.width(),
            height: monitor.height(),
            data: base64::encode(&buf),
            scale_factor: monitor.scale_factor(),
        });
    }
    Ok(captures)
}

/// 获取所有可见窗口列表（用于窗口识别）
#[tauri::command]
async fn list_windows() -> Result<Vec<WindowInfo>, String> {
    let windows = xcap::Window::all().map_err(|e| e.to_string())?;
    Ok(windows.iter()
        .filter(|w| !w.is_minimized() && w.width() > 0)
        .map(|w| WindowInfo {
            id: w.id(),
            title: w.title().to_string(),
            x: w.x(),
            y: w.y(),
            width: w.width(),
            height: w.height(),
            app_name: w.app_name().to_string(),
        })
        .collect())
}

/// 创建贴图窗口
#[tauri::command]
async fn create_pin_window(
    app: tauri::AppHandle,
    image_data: String,   // base64 PNG
    x: i32, y: i32,
    width: u32, height: u32,
) -> Result<String, String> {
    let label = format!("pin-{}", uuid::Uuid::new_v4());

    tauri::WebviewWindowBuilder::new(
        &app, &label,
        tauri::WebviewUrl::App("/screenshot-pin".into())
    )
    .title("")
    .inner_size(width as f64, height as f64)
    .position(x as f64, y as f64)
    .decorations(false)
    .transparent(true)
    .always_on_top(true)
    .skip_taskbar(true)
    .build()
    .map_err(|e| e.to_string())?;

    Ok(label)
}

/// 保存截图到文件
#[tauri::command]
async fn save_screenshot(path: String, data: Vec<u8>) -> Result<(), String> {
    std::fs::write(&path, &data).map_err(|e| e.to_string())
}
```

```javascript
  onMouseMove(e) { ... }  // 实时预览
  onMouseUp(e)   { ... }  // 完成绘制，推入 annotations

  // 撤销/重做
  undo() {
    if (!this.annotations.length) return
    this.redoStack.push(this.annotations.pop())
    this.redraw()
  }
  redo() {
    if (!this.redoStack.length) return
    this.annotations.push(this.redoStack.pop())
    this.redraw()
  }

  redraw() {
    this.ctx.drawImage(this.bgImage, 0, 0)
    for (const anno of this.annotations) {

    }
  }
```

## 七、注意事项

1. **多显示器**：xcap 支持多显示器枚举，需要正确处理坐标偏移和 DPI 缩放
2. **DPI 缩放**：Windows 高 DPI（125%/150%）下像素坐标需要乘以 scale_factor
3. **性能**：全屏截图 base64 传输较大（4K ≈ 10MB+），考虑用临时文件代替
4. **权限**：Windows 下截图无需特殊权限；macOS 需要「屏幕录制」权限
5. **透明窗口**：Windows 下透明窗口在某些显卡驱动下可能有兼容性问题
6. **退出清理**：ESC 键取消截图时需正确关闭覆盖窗口
