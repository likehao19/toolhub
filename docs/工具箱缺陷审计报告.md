## 一、安全漏洞

### P0 — CryptoTool XSS 注入

**文件**：`src/views/CryptoTool.vue:571-574`

`htmlDecode()` 使用 `div.innerHTML = str` 直接将用户输入赋值给 innerHTML，恶意 HTML/JS 会被浏览器解析执行。

```js
function htmlDecode(str) {
  const div = document.createElement('div')
  div.innerHTML = str  // ← XSS：用户输入直接插入 DOM
  return div.textContent || ''
}
```

**修复方案**：使用 DOMParser 替代，或纯字符串替换实体。

---

### P1 — EbookReader v-html 未充分转义

**文件**：`src/views/EbookReader.vue:37`

搜索摘要通过 `v-html="r.excerpt"` 渲染，`excerpt` 中包含高亮标签 `<mark>`，但如果书籍内容本身含有 HTML 标签，可能绕过 `escapeHtml`。

```html
<span class="search-excerpt" v-html="r.excerpt"></span>
```

**修复方案**：确认 `highlightSearchText` 函数先 escapeHtml 再插入 `<mark>` 标签。

---

### P2 — RegexTester ReDoS 无超时保护

**文件**：`src/views/RegexTester.vue:183-190`

用户输入的正则直接通过 `new RegExp()` 执行，虽有 10000 次循环限制，但灾难性回溯模式（如 `(a+)+$` 配合长字符串）仍可冻结 UI 数秒甚至更久。

```js
const re = new RegExp(pattern.value, flagsStr.value)
while ((m = re.exec(testText.value)) !== null && safety < 10000) { ... }
```

**修复方案**：将正则执行放入 Web Worker，或加入 `performance.now()` 超时中断（如 2 秒）。

---

### P2 — SqliteManager SQL 拼接风险

**文件**：`src/views/SqliteManager.vue:357`

表名通过字符串拼接构建 SQL，虽然表名来自 `sqliteTables()` API 返回而非直接用户输入，但双引号转义不完整（仅 `replace(/"/g, '""')`），如果表名含有特殊字符仍有风险。

```js
const tbl = currentTable.value.replace(/"/g, '""')
await sqliteQuery(connId.value, `SELECT * FROM "${tbl}" LIMIT ${pageSize.value} OFFSET ${offset}`, [])
```

**修复方案**：`pageSize` 和 `offset` 应使用参数化查询的绑定参数。

## 二、逻辑 Bug

### EbookShelf 搜索图标不显示

**文件**：`src/views/EbookShelf.vue:14`

```html
<el-input prefix-icon="Search" ... />
```

Element Plus 的 `prefix-icon` 需要传入组件引用而非字符串。字符串 `"Search"` 不会渲染任何图标。

**修复方案**：改为 `<template #prefix><el-icon><Search /></el-icon></template>`。

---

### 多工具 catch {} 吞掉错误

以下工具在 catch 块中不做任何错误反馈，用户操作失败时无感知：

| 文件 | 行号 | 场景 |
|------|------|------|
| `PortManager.vue` | 258 | 端口刷新失败 |
| `GitManager.vue` | 405, 474 | Git 操作失败 |
| `GitDailyReport.vue` | 273, 357, 368 | 扫描/导出失败 |
| `SdkManager.vue` | 178 | 目录选择失败 |
| `ScreenshotManager.vue` | 151 | 截图状态查询失败 |

**修复方案**：至少添加 `ElMessage.error(e?.message || String(e))`。

---

### CryptoTool 使用废弃 API

**文件**：`src/views/CryptoTool.vue:547`

```js
outputText.value = decodeURIComponent(escape(atob(text)))
```

`escape()` 是已废弃的全局函数，在严格模式或未来 JS 版本中可能被移除。对非 ASCII 字符的处理也不可靠。

**修复方案**：使用 `TextDecoder` + `Uint8Array` 替代。

---

### RegexTester computed 无防抖

**文件**：`src/views/RegexTester.vue:179`

`matches` 是 `computed` 属性，每次按键都会重新执行正则匹配。对于复杂正则 + 大文本，会导致严重输入卡顿。

**修复方案**：将 `matches` 改为 `ref` + `watch` + debounce（300ms）。

| `ApiDocs.vue` | 无清理 |
