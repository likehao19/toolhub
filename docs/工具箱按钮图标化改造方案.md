# 工具箱按钮图标化改造方案

> 范围：`/toolbox/*` 路由下 33 个工具页（不含 NoteView / AIChat / 等）。
> 目标：把高频通用动作（复制/删除/刷新/编辑/添加/关闭/设置）从「图标 + 文字」或「纯文字」改造为「纯图标 + tooltip」，对齐 CryptoTool / MultiPrint 风格，让顶栏 / 工具条 / 列表行更紧凑。
> 总体规模：约 228 个按钮，其中 ~35 个已是图标、~90 个可改纯图标、~70 个保留文字 + 加图标、~30 个 dialog footer 不改。

---

## 一、设计原则

### ✅ 改成纯图标按钮（icon-only）
满足以下**全部**条件：
1. **位置受限**：顶栏 / 工具条 / 列表行内 / 卡片操作区
2. **动作通用**：复制 / 删除 / 刷新 / 编辑 / 关闭 / 设置 / 上下移 / 添加 / 搜索 / 清空 / 重置
3. **图标语义自明**：用户一看图标就懂，不需要文字解释
4. **必须配 `:title`** 或 `el-tooltip` 提供无障碍信息

### ⚠️ 保留文字 + 加图标（icon + label）
满足以下**任一**条件：
1. **主操作按钮**（type="primary"）：开始查询 / 连接 / 发送 / 加密 / 格式化 / 生成报告 / 测试连接 / 一键打印
2. **状态切换文案需明示**："开始测试" vs "停止测试"、"连接" vs "断开"
3. **业务专有动作**：Pull / Push / Stash / Pop / SADD / RPUSH / 复制 Maven / 复制 Gradle 等

### ⛔ 不改（保持原状）
1. **dialog footer 的取消 / 确认 / 保存 / 关闭**（无障碍 + 用户习惯）
2. **el-radio-button**（语义是 radio 不是 button）
3. **tab 切换 / 标签按钮**
4. **空状态引导按钮**（首次进入页面提示用户的大按钮，需要文字传达）

## 三、标准代码模式

### 模式 A：顶栏 / 工具条的纯图标按钮（最常用）

```vue
<el-button size="small" text @click="doRefresh" :title="t('xxx.refresh')">
  <el-icon><Refresh /></el-icon>
</el-button>
```

要点：
- `size="small"` + `text` 两个 prop 配合给出"轻量化"视觉
- `:title` 用 i18n key（必加，无障碍 + hover 提示）
- 不再加 `<span>` 文字
- 多个连排时，外层加 gap：`<div style="display:flex;gap:4px">...</div>`

### 模式 B：列表行 / 卡片操作区的图标按钮

```vue
<el-button size="small" text type="danger" @click="onDelete(row)" :title="t('common.delete')">
  <el-icon><Delete /></el-icon>
</el-button>
```

要点：
- danger / primary / warning 用 `type` 表达色彩语义
- 列表行的图标按钮宽度受限，纯图标节省空间最明显

### 模式 C：主操作按钮（保留文字 + 加图标）

```vue
<el-button type="primary" :loading="loading" @click="doSubmit">
  <el-icon style="margin-right: 6px;"><Search /></el-icon>
  {{ t('xxx.startSearch') }}
</el-button>
```

要点：
- **保留文字** + 加图标视觉锚点
- `style="margin-right: 6px"` 是图标和文字的间距约定
- 主操作绝不改纯图标

### 模式 D：状态切换按钮

```vue
<!-- ✅ 推荐：图标 + 文字 -->
<el-button :type="running ? 'danger' : 'success'" @click="toggle">
  <el-icon style="margin-right: 6px;">
    <VideoPause v-if="running" /><VideoPlay v-else />
  </el-icon>
  {{ running ? t('mockService.stop') : t('mockService.start') }}
</el-button>
```

### 模式 E：图标按钮组（紧凑排列）

```vue
<div class="header-right">
  <el-button size="small" text @click="onSetting" :title="t('common.settings')">
    <el-icon><Setting /></el-icon>
  </el-button>
  <el-button size="small" text @click="onRefresh" :title="t('common.refresh')">
    <el-icon><Refresh /></el-icon>
  </el-button>
  <el-button size="small" text @click="onClear" :title="t('common.clear')">
    <el-icon><Delete /></el-icon>
  </el-button>
</div>
```

### 模式 F：i18n 兜底（避免出现裸中文 :title）

如果当前 i18n 文件没有对应 key，先去 `src/i18n/locales/zh-CN.js` 的 `common` 区段找：

```js
common: {
  save: '保存',
  cancel: '取消',
  delete: '删除',
  edit: '编辑',
  add: '添加',
  refresh: '刷新',
  copy: '复制',           // 如缺失需补
  clear: '清空',
  reset: '重置',
  close: '关闭',
  settings: '设置',       // 如缺失需补
  search: '搜索',
}
```

`:title` 一律用 `t('common.xxx')`，避免硬编码。

## 八、注意事项

1. **`text` 风格 vs 默认风格**：顶栏图标按钮统一用 `text`，看起来更轻量；列表行的 danger 操作按钮可保留默认风格。
2. **图标大小**：el-icon 默认继承父级 font-size，不需要额外指定。如需调整，统一在 scoped 样式里 `.xxx-btn .el-icon { font-size: 14px }`。
3. **加 :title 而不是 el-tooltip**：除非需要富文本提示，否则 `:title` 更轻量、不引入额外组件。
4. **danger 删除按钮**：`type="danger" text` 配合，hover 红色高亮提示破坏性操作。
5. **多语言**：`:title` 一律用 `t()`，**禁止**写死中文字符串。
6. **emoji 按钮全部替换**：📌 / ✕ / 🔗 / 🔓 / ‹ / › / × 这些字符按钮一律改成 element-plus 图标，保证字体不变情况下视觉一致。
