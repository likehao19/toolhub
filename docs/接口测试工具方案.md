# API 工作台方案（API Workbench）

> 集 **接口调试（Postman）+ 接口文档（Swagger）+ Mock 服务 + 性能测试（JMeter）** 于一体的一站式 API 开发工具，类似 Apifox。

## 一、功能全景

```
┌─────────────────────────────────────────────────────────┐
│                    API Workbench                        │
├─────────┬─────────┬─────────┬─────────┬────────────────┤
│  接口调试  │ 接口文档  │ Mock 服务 │ 性能测试  │  自动化测试   │
│ Postman  │ Swagger │  Mock   │ JMeter  │  Test Runner  │
├─────────┼─────────┼─────────┼─────────┼────────────────┤
│ 请求构建  │ API 定义 │ 智能 Mock │ 并发测试  │  断言脚本     │
│ 响应展示  │ 数据模型 │ 自定义规则│ 压力测试  │  前/后置脚本   │
│ 环境变量  │ 在线文档 │ Mock 服务 │ 测试报告  │  批量运行     │
│ 请求历史  │ 导入导出 │ 期望匹配  │ 可视化图表│  CI/CD 导出   │
│ 集合管理  │ 变更日志 │          │         │              │
└─────────┴─────────┴─────────┴─────────┴────────────────┘
```

| 模块 | 功能 | 说明 |
|------|------|------|
| **接口调试** | 请求构建 | GET/POST/PUT/DELETE/PATCH/HEAD/OPTIONS，支持 {{变量}} |
| | 请求参数 | Params / Headers / Body(JSON/Form/XML/Binary/GraphQL) / Auth / Cookie |
| | 响应展示 | Body(Pretty/Raw/Preview) / Headers / Cookies / Timeline |
| | 请求历史 | 自动保存，按时间分组，快速回填 |
| | 集合管理 | 文件夹/分组，拖拽排序，导入导出 |
| | 环境变量 | 多环境切换，全局/环境/临时变量 |
| | cURL 导入 | 粘贴 cURL 命令自动解析为请求 |
| **接口文档** | API 定义 | 可视化表单定义接口（路径/方法/参数/响应结构） |
| | 数据模型 | 定义可复用的 JSON Schema 数据结构 |
| | 文档预览 | 渲染为可读的 API 文档页面 |
| | 导入导出 | 支持 OpenAPI 3.0 / Swagger 2.0 / Postman Collection 导入导出 |
| | 变更日志 | 自动记录接口修改历史 |
| **Mock 服务** | 智能 Mock | 根据字段名/类型自动生成模拟数据 |
| | 自定义规则 | Mock.js 语法规则（`@name`、`@email`、`@image`） |
| | 期望匹配 | 根据请求参数返回不同 Mock 响应 |
| | Mock 服务器 | 本地启动 HTTP Mock 服务，前端可直接调用 |
| **性能测试** | 并发测试 | 设置并发数 + 持续时间，批量发送请求 |
| | 压力测试 | 阶梯递增并发，找到接口瓶颈 |
| | 测试报告 | QPS / 平均响应时间 / P99 / 错误率 / 成功率 |
| | 可视化图表 | 实时折线图（响应时间、QPS、错误率随时间变化） |
| **自动化测试** | 断言脚本 | 状态码/Header/Body 字段断言 |
| | 前/后置脚本 | 请求前设置变量、请求后提取数据（如提取 token） |
| | 批量运行 | 选择集合一键跑全部接口，生成测试报告 |
| | 测试套件 | 多接口编排成测试流程，上下文传递 |

## 三、核心功能详细设计

### 3.1 接口调试（Postman 功能）

#### 3.1.1 请求构建

```
方法选择器（带颜色标签）：
  GET（绿色 #67C23A）/ POST（橙色 #E6A23C）/ PUT（蓝色 #409EFF）
  / DELETE（红色 #F56C6C）/ PATCH（紫色 #a855f7）/ HEAD / OPTIONS

URL 输入框：
  - 支持 {{变量}} 语法，高亮显示变量，hover 预览实际值
  - 输入时自动补全历史 URL
  - URL 解析：自动提取 query params 填入 Params Tab

发送按钮：
  - 点击或 Ctrl+Enter 发送
  - 发送中显示 loading + 取消按钮（AbortController）
  - 下拉菜单：发送并下载 / 生成代码 / 复制 cURL
  - 支持请求超时设置（默认 30s）
```

#### 3.1.2 请求参数 Tabs

**Params Tab**
- Key-Value 表格，每行有启用/禁用 checkbox
- 与 URL query string 双向同步
- 批量编辑模式（切换为文本 `key=value\n` 格式）
- Path 变量支持（`/users/:id` → 自动提取 `:id` 为可编辑字段）

**Headers Tab**
- 预置常用 Header（Content-Type / Authorization / Accept / User-Agent）
- Key 输入自动补全标准 HTTP Header
- 显示自动追加的 Header（灰色，如 Content-Length）

**Body Tab**
- 类型切换：`none` | `JSON` | `form-data` | `x-www-form-urlencoded` | `raw` | `binary` | `GraphQL`
- **JSON**：CodeMirror 编辑器，语法高亮 + 格式化 + 校验
- **form-data**：Key-Value 表格，Value 类型支持 Text / File
- **GraphQL**：Query 编辑器 + Variables 编辑器

**Auth Tab**
- `None` | `Bearer Token` | `Basic Auth` | `API Key` | `OAuth 2.0`
- **Bearer Token**：输入框 + 变量引用 `{{token}}`
- **Basic Auth**：用户名 + 密码 → 自动生成 Base64
- **API Key**：Key + Value + 位置（Header / Query Param）
- **OAuth 2.0**：Grant Type / Auth URL / Token URL / Client ID / Secret

**Pre-request Script Tab（前置脚本）**
```javascript
// 可用 API
pm.variables.set('timestamp', Date.now())
pm.variables.set('sign', md5(body + secret))
pm.environment.get('baseUrl')
```

**Post-response Script Tab（后置脚本）**
```javascript
// 自动提取 token
const data = pm.response.json()
if (data.code === 0) {
  pm.variables.set('token', data.data.token)
}
```

**Tests Tab（断言）**
```javascript
// 可视化断言构建器 + 脚本模式
pm.test('状态码为200', () => pm.response.status === 200)
pm.test('返回成功', () => pm.response.json().code === 0)
pm.test('响应时间<500ms', () => pm.response.time < 500)
```

#### 3.1.3 响应展示

```
状态栏：[200 OK](绿色)  Time: 123ms  Size: 2.1KB  断言: 3/3 ✅

Body Tab：
  - Pretty：JSON 语法高亮 + 可折叠/展开节点 + 复制节点路径
  - Raw：原始文本
  - Preview：HTML/Image 渲染预览
  - 可视化：JSON 树形视图（类似浏览器 DevTools）
  - 底部工具栏：搜索 / 复制 / 保存到文件 / 自动换行

Headers Tab：响应头 Key-Value 列表
Cookies Tab：Cookie 详情（名称/值/域/路径/过期/HttpOnly/Secure）

Timeline Tab（时间线）：
  - DNS Lookup: 5ms
  - TCP Connection: 12ms
  - TLS Handshake: 23ms
  - Request Sent: 1ms
  - Waiting (TTFB): 89ms
  - Content Download: 3ms
  - Total: 133ms
```

#### 3.1.4 请求历史

- 自动保存每次请求（方法/URL/参数/响应状态/时间戳）
- 按时间分组显示（今天 / 昨天 / 本周 / 更早）
- 点击回填 + 重发
- 搜索过滤（URL / 方法 / 状态码）
- 批量删除 / 清空

#### 3.1.5 cURL 导入

```
支持粘贴 cURL 命令，自动解析为：
  - 方法、URL
  - Headers（-H / --header）
  - Body（-d / --data / --data-raw）
  - Auth（-u / --user）
  - Cookies（-b / --cookie）

示例输入：
  curl -X POST https://api.example.com/login \
    -H 'Content-Type: application/json' \
    -d '{"username":"admin","password":"123"}'
```

#### 3.1.6 代码生成

从当前请求生成多语言代码：
- JavaScript (fetch / axios)
- Python (requests)
- Java (OkHttp)
- Go (net/http)
- cURL
- PHP (curl)

### 3.2 接口文档（Swagger 功能）

#### 3.2.1 API 定义（可视化表单）

```javascript
// 接口定义数据结构
{
  id: 'uuid',
  name: '创建用户',
  method: 'POST',
  path: '/api/users',
  group: '用户管理',
  tags: ['用户', '管理后台'],
  description: '创建一个新用户',

  // 请求定义
  request: {
    headers: [...],
    params: {
      query: [...],
      path: [...]
    },
    body: {
      type: 'application/json',
      schema: { $ref: '#/models/CreateUserReq' }
    }
  },

  // 响应定义（多状态码）
  responses: [
    {
      statusCode: 200,
      description: '成功',
      schema: { $ref: '#/models/User' }
    },
    {
      statusCode: 400,
      description: '参数错误',
      schema: { type: 'object', properties: { code: {}, message: {} } }
    }
  ]
}
```

#### 3.2.2 数据模型

```javascript
// 可复用的数据模型定义
{
  id: 'uuid',
  name: 'User',
  description: '用户模型',
  schema: {
    type: 'object',
    required: ['id', 'username', 'email'],
    properties: {
      id:        { type: 'integer', description: '用户ID', example: 1 },
      username:  { type: 'string',  description: '用户名', example: 'tom' },
      email:     { type: 'string',  format: 'email', description: '邮箱' },
      avatar:    { type: 'string',  format: 'uri', description: '头像URL' },
      roles:     { type: 'array',   items: { type: 'string' }, description: '角色列表' },
      createdAt: { type: 'string',  format: 'date-time', description: '创建时间' }
    }
  }
}
```

- 可视化 Schema 编辑器：表格形式编辑字段（名称/类型/必填/示例/描述）
- 支持嵌套结构（对象/数组/引用其他模型 `$ref`）
- 从 JSON 示例自动生成 Schema

#### 3.2.3 文档预览

- 渲染为可读的 API 文档页面，样式类似 Swagger UI

### 5.4 HTTP 请求方案

优先使用 `@tauri-apps/plugin-http`（Rust reqwest，完全无跨域限制）：

```javascript
import { fetch } from '@tauri-apps/plugin-http'

export async function sendRequest(config) {
  const startTime = performance.now()

  const response = await fetch(config.url, {
    method: config.method,
    headers: config.headers,
    body: config.body,
    connectTimeout: config.timeout
  })

  const body = await response.text()

  return {
    status: response.status,
    statusText: response.statusText,
    headers: Object.fromEntries(response.headers.entries()),
    body,
    time: Math.round(performance.now() - startTime),
    size: new Blob([body]).size
  }
}
```
