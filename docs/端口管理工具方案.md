# 端口管理工具方案

## 概述

在工具箱中新增「端口管理」工具，支持按端口号或 PID 查询进程占用情况，并可一键终止占用端口的进程。兼容 Windows 和 macOS 双系统。

## 功能需求

| 功能 | 说明 |
|------|------|
| 按端口查询 | 输入端口号，查询占用该端口的进程信息（PID、进程名、协议、状态） |
| 按 PID 查询 | 输入 PID，查询该进程占用的所有端口 |
| 查看全部端口 | 列出当前所有活跃的端口连接（支持筛选 TCP/UDP、LISTENING 等状态） |
| 终止进程 | 选中端口后可终止对应进程，操作前二次确认 |
| 刷新 | 手动刷新端口列表 |

## 技术方案

### 系统命令

#### Windows

```bash
# 查看所有端口
netstat -ano

# 按端口查询（筛选输出）
netstat -ano | findstr :8080

# 按 PID 查询
netstat -ano | findstr /E " 1234"

# 获取进程名（通过 PID）
tasklist /FI "PID eq 1234" /FO CSV /NH

# 终止进程
taskkill /PID 1234 /F
```

#### macOS

```bash
# 查看所有端口
lsof -iTCP -sTCP:LISTEN -P -n
netstat -anv

# 按端口查询
lsof -iTCP:8080 -P -n

# 按 PID 查询
lsof -p 1234 -iTCP -P -n

# 获取进程名（通过 PID）
ps -p 1234 -o comm=

# 终止进程
kill -9 1234
```

### 命令输出解析

#### Windows `netstat -ano` 输出格式

```
协议    本地地址          外部地址          状态           PID
TCP    0.0.0.0:135       0.0.0.0:0        LISTENING      1056
TCP    127.0.0.1:8080    127.0.0.1:52341  ESTABLISHED    12345
UDP    0.0.0.0:5353      *:*                             2468
```

解析策略：按行分割，跳过表头，用正则提取各字段：
```
/^\s*(TCP|UDP)\s+([\d.:]+)\s+([\d.:*]+)\s*(\w*)\s+(\d+)\s*$/
```

#### macOS `lsof -iTCP -P -n` 输出格式

```
COMMAND   PID   USER   FD   TYPE   DEVICE SIZE/OFF NODE NAME
node     1234   user   23u  IPv4   0x...  0t0      TCP  *:3000 (LISTEN)
```

解析策略：按空格分割，提取 COMMAND、PID、NAME 字段。

### 数据模型

```typescript
interface PortInfo {
  protocol: 'TCP' | 'UDP'       // 协议
  localAddress: string           // 本地地址
  localPort: number              // 本地端口
  remoteAddress: string          // 远程地址
  remotePort: number | null      // 远程端口
  state: string                  // 连接状态 (LISTENING, ESTABLISHED, TIME_WAIT, etc.)
  pid: number                    // 进程 PID
  processName: string            // 进程名称
}
```

### 交互细节

- **搜索框**：支持输入端口号或 PID，自动识别（纯数字 ≤ 65535 视为端口，否则视为 PID），也可手动切换模式
- **协议筛选**：下拉选择 全部 / TCP / UDP
- **状态筛选**：可选筛选 LISTENING / ESTABLISHED / TIME_WAIT / CLOSE_WAIT 等
- **终止按钮**：点击后弹出 `ElMessageBox.confirm` 确认，显示"确定终止进程 {processName} (PID: {pid}) 吗？"
- **表格**：使用 `el-table` 展示，支持排序（按端口号、PID）
- **刷新**：手动刷新按钮 + 首次进入自动加载

## 核心逻辑（portManager.js）

```javascript
import { executeCommand } from '@/utils/tauri/shell.js'

const isWindows = navigator.userAgent.includes('Windows')

// 获取全部端口列表
async function getAllPorts() {
  if (isWindows) {
    const { stdout } = await executeCommand('netstat', ['-ano'])
    return parseWindowsNetstat(stdout)
  } else {
    const { stdout } = await executeCommand('lsof', ['-iTCP', '-iUDP', '-P', '-n'])
    return parseMacLsof(stdout)
  }
}

// 为每条记录补充进程名
async function getProcessName(pid) {
  if (isWindows) {
    const { stdout } = await executeCommand('tasklist', ['/FI', `PID eq ${pid}`, '/FO', 'CSV', '/NH'])
    // 解析 CSV: "进程名","PID","会话名","会话#","内存使用"
    return stdout.split(',')[0]?.replace(/"/g, '').trim()
  } else {
    const { stdout } = await executeCommand('ps', ['-p', String(pid), '-o', 'comm='])
    return stdout.trim()
  }
}

// 终止进程
async function killProcess(pid) {
  if (isWindows) {
    return executeCommand('taskkill', ['/PID', String(pid), '/F'])
  } else {
    return executeCommand('kill', ['-9', String(pid)])
  }
}
```

## 注意事项

1. **权限**：Windows 下 `taskkill` 终止系统进程可能需管理员权限，终止失败时需友好提示
2. **编码**：Windows `netstat` 输出可能为 GBK，shell.js 中已通过 Rust 端处理编码转换
3. **性能**：`netstat -ano` 输出可能有数百行，一次性加载即可，无需分页
4. **安全**：终止进程前必须二次确认，防止误杀关键系统进程
5. **PID 0/4**：Windows 系统进程（PID 0 Idle、PID 4 System）不应允许终止
