# 文件比对工具方案

## 一、功能概述

| 能力 | 说明 |
|------|------|
| 双栏对比 | 左右分栏显示两个文件，差异行高亮标记（新增/删除/修改） |
| 内联对比 | 单栏统一视图，用 `+/-` 标记变更，类似 `git diff` |
| 行内差异 | 同一行中精确标记变更的字符片段（字符级 diff） |
| 在线编辑 | 两侧编辑器均可直接编辑，实时重新计算差异 |
| 合并操作 | 点击箭头将某一侧的变更合并到另一侧（← / →） |
| 差异导航 | 上一处/下一处差异跳转，显示当前位置 `3/12` |
| 文件加载 | 支持拖拽文件、点击选择文件、粘贴文本三种方式 |
| 保存导出 | 保存编辑后的文件，或导出差异报告（`.diff` / `.html`） |
| 语法高亮 | 根据文件扩展名自动识别语言并应用 CodeMirror 语法高亮 |
| 编码支持 | 自动检测 UTF-8 / GBK 等常见编码 |

## 二、技术选型

### Diff 引擎

| 方案 | 包大小 | 行级 diff | 字符级 diff | 性能 | 选择 |
|------|--------|----------|------------|------|------|
| **diff** (npm) | 12 KB | ✅ `diffLines` | ✅ `diffChars` | 优 | ✅ |
| diff-match-patch | 30 KB | ✅ | ✅ | 优 | ❌ API 较老 |
| jsdiff (同 diff) | — | — | — | — | 同上 |

选择 **`diff`**（即 jsdiff），理由：
- 轻量，API 简洁（`diffLines`、`diffChars`、`createPatch`）
- 项目已有 CodeMirror 基础，无额外学习成本
- 支持直接生成 unified diff 格式，方便导出

### 编辑器

复用项目已有的 **CodeMirror 6**，与 CodeFormatter 保持一致。

### 文件读写

通过 Tauri API（`@tauri-apps/plugin-dialog` + `@tauri-apps/plugin-fs`）实现本地文件读写。

### 视图模式

**1. 并排模式（Side-by-Side）**
- 左右两个 CodeMirror 编辑器，同步滚动
- 删除行：左侧红色背景 `rgba(255,0,0,0.08)`，行号旁 `-` 标记
- 新增行：右侧绿色背景 `rgba(0,255,0,0.08)`，行号旁 `+` 标记
- 修改行：两侧黄色背景 `rgba(255,165,0,0.08)`，变更字符加深高亮
- 中间连接线：SVG 曲线连接对应差异块
- 合并箭头：`→` 将左侧内容合并到右侧，`←` 反向

**2. 内联模式（Inline / Unified）**
- 单个编辑器，删除行红色 + 新增行绿色交替显示
- 行首 `- / +` 前缀
- 折叠无变更区域（显示 `··· 15 行未变更 ···`）

### 4.3 DiffEditor.vue — 编辑器组件

关键能力：

| 功能 | 实现方式 |
|------|---------|
| 差异行高亮 | CodeMirror `Decoration.line()` + `StateField` |

| 实时 diff | `EditorView.updateListener` 监听内容变化，debounce 300ms 重算 |

### 4.4 同步滚动算法

```
并排模式下，两侧行数可能不同（插入/删除导致偏移）。
使用差异块映射表维护行号对应关系：

  左侧行号 → 差异块 → 右侧行号

滚动时：
  1. 获取当前可见区域首行号
  2. 通过映射表查找对侧对应行
  3. 计算对侧 scrollTop = 对应行 × lineHeight
  4. 设置对侧滚动位置（加锁防止循环触发）
```

### 4.5 文件读取与编码

```javascript
import { open } from '@tauri-apps/plugin-dialog'
import { readFile, writeFile } from '@tauri-apps/plugin-fs'

async function openFile(side) {
  const path = await open({
    filters: [{ name: '所有文件', extensions: ['*'] }]
  })
  if (!path) return

  const bytes = await readFile(path)
  // 尝试 UTF-8 解码，失败则尝试 GBK
  let text
  try {
    text = new TextDecoder('utf-8', { fatal: true }).decode(bytes)
  } catch {
    text = new TextDecoder('gbk').decode(bytes)
  }

  return { text, path, encoding: detectedEncoding }
}
```

## 七、交互细节

1. **拖拽加载**：左半区拖入 → 加载到左侧编辑器，右半区同理
2. **快捷键**：
   - `Alt+↑` / `Alt+↓` — 上一处 / 下一处差异
   - `Ctrl+S` — 保存当前焦点侧文件
   - `Ctrl+Shift+S` — 同时保存两侧
   - `Alt+→` — 将当前差异合并到右侧
   - `Alt+←` — 将当前差异合并到左侧
3. **自动检测语言**：根据文件扩展名匹配 CodeMirror 语言包（`.js` → JavaScript, `.py` → Python 等）
4. **大文件保护**：文件 > 5MB 时提示警告，> 20MB 拒绝加载
5. **修改标记**：文件名旁显示 `●` 表示有未保存修改
