# 首页实时新闻模块方案

> 目标：在首页 Dashboard 新增一个「实时新闻」卡片，展示热点新闻，支持分类切换与点击阅读。

---

## 一、免费新闻 API 调研

### 1.1 推荐方案对比

| API | 免费额度 | 中文支持 | 需要Key | 延迟 | 推荐指数 |
|-----|---------|---------|---------|------|---------|
| **Google News RSS** | **无限制** | 原生中文 | **不需要** | 实时 | ⭐⭐⭐⭐⭐ |
| **Currents API** | 1000次/天 | 78种语言含中文 | 需要（免费） | 实时 | ⭐⭐⭐⭐ |
| **天聚数行 TianAPI** | 100次/天 | 原生中文 | 需要（免费） | 实时 | ⭐⭐⭐⭐ |
| **GNews API** | 100次/天 | Google News 中文 | 需要（免费） | 实时 | ⭐⭐⭐ |
| **NewsAPI.org** | 100次/天 | 14种语言 | 需要（免费） | **延迟24h** | ⭐⭐ |
| **NewsData.io** | 200次/天 | 多语言 | 需要（免费） | 实时 | ⭐⭐⭐ |

### 1.2 最终推荐：Google News RSS（主） + Currents API（备）

**推荐理由：**

#### 主方案 — Google News RSS

- **完全免费，无需 API Key，无请求次数限制**
- 原生支持中文（`hl=zh-CN&gl=CN&ceid=CN:zh-Hans`）
- 支持按分类获取（科技、财经、体育、娱乐等）
- 返回 XML/RSS，前端用 `DOMParser` 或后端用 Rust `feed-rs` 解析
- Google News 覆盖面广，数据质量高

```
# 中文头条
https://news.google.com/rss?hl=zh-CN&gl=CN&ceid=CN:zh-Hans

# 分类新闻（科技）
https://news.google.com/rss/topics/CAAqJggKIiBDQkFTRWdvSUwyMHZNRGRqTVhZU0FucG9MWE5uR0FBUAE?hl=zh-CN&gl=CN&ceid=CN:zh-Hans

# 关键词搜索
https://news.google.com/rss/search?q=人工智能&hl=zh-CN&gl=CN&ceid=CN:zh-Hans
```

#### 备用方案 — Currents API

- 1000 次/天免费额度充裕
- REST JSON 接口，无需解析 XML
- 覆盖 70+ 国家、78 种语言

```
GET https://api.currentsapi.services/v1/latest-news?language=zh&apiKey=YOUR_KEY
GET https://api.currentsapi.services/v1/search?keywords=科技&language=zh&apiKey=YOUR_KEY
```

### 3.2 数据流

```
Google News RSS (XML)
       │
       ▼
Rust 后端 (src-tauri/src/commands/news.rs)
  ├─ reqwest 请求 RSS XML
  ├─ feed-rs / quick-xml 解析
  ├─ 结构化为 Vec<NewsItem>
  └─ 内存缓存（HashMap + 过期时间）
       │
       ▼
Tauri Command: fetch_news(category)
       │
       ▼
Vue 前端 (invoke → JSON)
  ├─ NewsCard.vue 展示
  ├─ 分类 Tab 切换
  └─ 点击 → shell.open(url) 打开浏览器
```

**为什么用 Rust 后端请求而不是前端直接 fetch：**

1. **跨域问题**：浏览器/WebView 无法直接请求 `news.google.com` 的 RSS（CORS 限制）
2. **XML 解析**：Rust 的 `quick-xml` / `feed-rs` 性能远优于前端 JS 解析
3. **缓存控制**：Rust 端做内存缓存，避免重复请求，减少延迟
4. **网络代理**：Tauri 端可配置系统代理，解决国内访问 Google News 的问题

### 3.3 RSS 返回数据结构（解析后）

```rust
#[derive(Serialize, Clone)]
pub struct NewsItem {
    pub title: String,        // 新闻标题
    pub link: String,         // 原文链接
    pub source: String,       // 来源（新华网、人民日报...）
    pub pub_date: String,     // 发布时间 ISO 8601
    pub description: String,  // 摘要（可能含 HTML）
    pub image_url: Option<String>,  // 缩略图（如有）
}
```

Google News RSS `<item>` 示例：

```xml
<item>
  <title>新闻标题 - 来源名</title>
  <link>https://news.google.com/rss/articles/...</link>
  <pubDate>Thu, 20 Mar 2026 08:30:00 GMT</pubDate>
  <description>&lt;a href="..."&gt;...&lt;/a&gt;</description>
  <source url="https://www.xinhuanet.com">新华网</source>
</item>
```

## 五、缓存与刷新策略

| 策略 | 说明 |
|------|------|
| Rust 内存缓存 | `HashMap<category, (data, timestamp)>`，TTL 10 分钟 |
| 首页加载 | `onMounted` 时请求当前分类新闻 |
| 切换分类 | 优先读缓存，过期则重新请求 |
| 手动刷新 | 点击刷新按钮强制忽略缓存 |
| 定时刷新 | 可选：`setInterval` 每 10 分钟自动更新 |
| 备用降级 | Google News 不可用时自动切换 Currents API |

## 六、国内访问 Google News 的处理

Google News 在国内可能需要代理才能访问，方案如下：

| 方案 | 说明 | 实现难度 |
|------|------|---------|
| **系统代理** | Rust `reqwest` 默认走系统代理，用户开 VPN 即可 | 低 |
| **设置页代理配置** | Settings 中添加 HTTP Proxy 输入框，传给 reqwest | 中 |
| **备用 API 自动降级** | 检测 Google 不可达则切 Currents / TianAPI | 中 |
| **RSS 中转** | 部署一个 Cloudflare Worker 中转 RSS | 中 |

**建议组合**：系统代理 + 备用 API 自动降级。

降级逻辑伪代码：

```rust
async fn fetch_news(category: &str) -> Result<Vec<NewsItem>> {
    // 尝试 Google News RSS
    match fetch_google_rss(category).await {
        Ok(items) if !items.is_empty() => return Ok(items),
        _ => {}
    }
    // 降级到 Currents API
    match fetch_currents_api(category).await {
        Ok(items) => return Ok(items),
        Err(e) => Err(format!("所有新闻源不可用: {}", e))
    }
}
```
