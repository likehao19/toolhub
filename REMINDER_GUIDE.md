# 提醒功能使用指南

## 概述

本应用的提醒功能分为两个部分：
- **待办提醒**：在待办管理中设置，可在开始日期、截止日期或过期时提醒
- **日程提醒**：在日程管理中设置，可设置提前提醒时间和重复规则

所有提醒会在桌面窗口的右下角（或您配置的其他位置）弹出自定义通知。

## 通知显示设置

在 **设置** → **提醒设置** 中配置通知的显示方式：

### 位置类型
- **软件内位置**：通知相对于应用窗口显示
- **桌面窗口位置**：通知相对于整个屏幕显示（推荐）

### 显示位置
可选择9个位置：
- 右上角、左上角、上中
- 右下角、左下角、下中
- 右中、左中、中心

### 测试功能
点击"测试通知"按钮可立即发送一条测试通知，查看效果。

## 待办提醒

### 在待办中设置提醒

1. 打开 **待办管理** 页面
2. 创建新待办或编辑现有待办
3. 在表单中找到"提醒"部分
4. 启用"提醒"开关
5. 选择**提醒类型**：
   - **开始日期提醒**：在开始日期当天提醒
   - **截止日期提醒**：在截止日期当天提醒
   - **截止前提醒**：在截止日期前N天提醒
   - **过期提醒**：如果过期，每天提醒
6. 设置**提醒时间**（例如 09:00）：每天在此时间检查并提醒

### 提醒触发逻辑

- 系统每60秒检查一次待办提醒
- 只有启用提醒且未完成的待办才会触发提醒
- 每个待办每天只提醒一次（在设定的时间点）
- 提醒时间允许±5分钟的误差范围

### 示例场景

**场景1：项目开始提醒**
```
标题：开始开发新功能
开始日期：2026-02-05
提醒：启用
提醒类型：开始日期提醒
提醒时间：09:00
```
→ 在 2026-02-05 的早上 9:00 会收到提醒

**场景2：任务截止提醒**
```
标题：提交月度报告
截止日期：2026-02-10
提醒：启用
提醒类型：截止日期提醒
提醒时间：14:00
```
→ 在 2026-02-10 的下午 2:00 会收到提醒

**场景3：过期任务提醒**
```
标题：完成代码审查
截止日期：2026-02-01（已过期）
提醒：启用
提醒类型：过期提醒
提醒时间：09:00
```
→ 每天早上 9:00 都会提醒任务已过期

## 日程提醒

### 在日程中设置提醒

1. 打开 **日程管理** 页面
2. 创建新日程或编辑现有日程（双击日期快速创建）
3. 在表单中找到"提醒"部分
4. 勾选需要的提醒时间：
   - 5分钟前
   - 15分钟前
   - 30分钟前
   - 1小时前
   - 1天前
5. 可以同时选择多个提醒时间

### 设置重复日程

在创建/编辑日程时：

1. 在"重复"下拉菜单中选择重复类型：
   - **不重复**：单次日程
   - **每天**：每天重复
   - **每周**：每周重复（可选择星期几）
   - **每月**：每月重复
   - **每年**：每年重复

2. 设置间隔（可选）：
   - 例如间隔为2的"每周"表示每两周重复一次

3. 对于每周重复，选择星期几：
   - 可以勾选多个工作日

### 提醒触发逻辑

- 系统每60秒检查一次日程提醒
- 检查未来2小时内的日程
- 在设定的提醒时间点（允许±5分钟误差）弹出通知
- 每个提醒时间点只提醒一次

### 示例场景

**场景1：会议提醒（多个提醒）**
```
标题：项目评审会议
开始时间：2026-02-05 14:00
地点：会议室A
提醒：15分钟、5分钟
```
→ 会在 13:45 和 13:55 分别收到提醒

**场景2：每周例会（重复提醒）**
```
标题：周例会
开始时间：2026-02-03 10:00
重复：每周
星期：周一
提醒：15分钟
```
→ 每周一 9:45 收到提醒

**场景3：每工作日站会（重复提醒）**
```
标题：每日站会
开始时间：2026-02-03 09:30
重复：每周
星期：周一、周二、周三、周四、周五
提醒：5分钟
```
→ 每个工作日 9:25 收到提醒

## 技术实现说明

### 提醒服务

提醒功能由 `reminderService.js` 提供：

- **自动初始化**：应用启动时自动启动提醒服务
- **定时检查**：每60秒检查一次待办和日程提醒
- **防重复机制**：使用 Set 记录已提醒的项目，避免重复提醒
- **配置持久化**：通知显示设置保存在 localStorage 中

### 数据库结构

**待办表 (todos)**
- `reminder_enabled`: 是否启用提醒 (0/1)
- `reminder_type`: 提醒类型 (on_start/on_due/before_due/overdue)
- `reminder_time`: 提醒时间 (HH:mm 格式)
- `reminder_sent`: 提醒是否已发送 (0/1)

**日程表 (calendar_events)**
- `reminder_minutes`: 提前提醒分钟数（向后兼容）
- `reminder_rules`: 提醒规则 JSON 数组（例如 [5, 15, 30]）
- `repeat_rule`: 重复规则字符串
- `repeat_end_date`: 重复结束日期

### 自定义通知

使用 Tauri 创建自定义通知窗口：
- 支持标题、消息、类型（info/warning/error/success）
- 可配置显示位置和时长
- 自动淡入淡出动画

## 常见问题

**Q: 为什么没有收到提醒？**
A: 请检查：
1. 提醒是否已启用
2. 应用是否正在运行
3. 提醒时间是否正确设置
4. 待办是否已完成（已完成的待办不提醒）

**Q: 可以自定义提醒声音吗？**
A: 当前版本暂不支持提醒声音，但可以配置通知的显示位置和样式。

**Q: 提醒会持续显示多久？**
A: 
- 待办提醒：15秒
- 日程提醒：20秒

**Q: 如何关闭某个提醒？**
A: 编辑对应的待办或日程，关闭"提醒"开关即可。

**Q: 重复日程的提醒如何工作？**
A: 重复日程在每次出现时都会按照设定的提醒规则触发提醒。

## 最佳实践

1. **合理设置提醒时间**
   - 重要任务设置多个提醒时间点
   - 日常任务选择合适的提醒时间（如早上9点）

2. **使用重复规则**
   - 每周例会、每日站会等固定日程使用重复规则
   - 减少重复创建日程的工作量

3. **优先级配合提醒**
   - 高优先级待办设置提前提醒
   - 低优先级待办设置截止日期提醒即可

4. **定期清理**
   - 完成待办后及时标记完成
   - 清理过期且不需要的日程
